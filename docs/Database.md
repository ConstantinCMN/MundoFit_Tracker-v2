# MundoFit Tracker V2 — Database

## Overview

The database is hosted on Supabase (managed Postgres). Every table has Row Level Security (RLS) enabled. All timestamps are stored in UTC (`timestamptz`). All weight values are stored in **kg**. All height values are stored in **cm**. Conversion to imperial units is done at the display layer only.

---

## Conventions

- Primary keys: `uuid` generated by `gen_random_uuid()`.
- Foreign key to auth user: `user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`.
- Soft deletes are **not** used. Hard deletes with undo are handled at the application layer (30-second undo window via optimistic UI before the delete is committed).
- `created_at` and `updated_at` on every table, with an auto-update trigger on `updated_at`.

## Tables

### `profiles`
Single source of truth for user data. One row per authenticated user, created automatically via a Supabase trigger on `auth.users` insert.

```sql
CREATE TABLE profiles (
  id                   uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id              uuid NOT NULL UNIQUE REFERENCES auth.users(id) ON DELETE CASCADE,

  -- Identity
  first_name           text,
  display_name         text,
  avatar_url           text,

  -- Onboarding fields
  gender               text CHECK (gender IN ('male', 'female')),
  age                  smallint CHECK (age BETWEEN 10 AND 120),
  height_cm            numeric(5,1) CHECK (height_cm BETWEEN 50 AND 300),
  weight_kg            numeric(5,2) CHECK (weight_kg BETWEEN 20 AND 500),
  activity_level       text CHECK (activity_level IN (
                         'sedentary', 'lightly_active', 'moderately_active',
                         'very_active', 'athlete'
                       )),
  training_location    text CHECK (training_location IN ('gym', 'home', 'both')),
  goal                 text CHECK (goal IN (
                         'lose_weight', 'build_muscle', 'improve_endurance',
                         'stay_healthy', 'athletic_performance'
                       )),
  onboarding_completed boolean NOT NULL DEFAULT false,

  -- Preferences
  locale               text NOT NULL DEFAULT 'ro' CHECK (locale IN ('ro', 'en', 'es')),
  unit_system          text NOT NULL DEFAULT 'metric' CHECK (unit_system IN ('metric', 'imperial')),

  created_at           timestamptz NOT NULL DEFAULT now(),
  updated_at           timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX idx_profiles_user_id ON profiles(user_id);
```

**RLS:**
```sql
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
CREATE POLICY "profiles_owner" ON profiles
  USING (user_id = auth.uid())
  WITH CHECK (user_id = auth.uid());
```

**Storage Policy (bucket `progress-photos`):**
```sql
-- Upload
CREATE POLICY "users_upload_own_photos" ON storage.objects
  FOR INSERT WITH CHECK (
    bucket_id = 'progress-photos' AND
    (storage.foldername(name))[1] = auth.uid()::text
  );

-- Read
CREATE POLICY "users_read_own_photos" ON storage.objects
  FOR SELECT USING (
    bucket_id = 'progress-photos' AND
    (storage.foldername(name))[1] = auth.uid()::text
  );

-- Delete
CREATE POLICY "users_delete_own_photos" ON storage.objects
  FOR DELETE USING (
    bucket_id = 'progress-photos' AND
    (storage.foldername(name))[1] = auth.uid()::text
  );
```

### `exercises`
Global exercise library. Populated by seeded data; users can also add custom exercises.

```sql
CREATE TABLE exercises (
  id               uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name_ro          text NOT NULL,
  name_en          text NOT NULL,
  name_es          text NOT NULL,
  description_ro   text,
  description_en   text,
  description_es   text,
  muscle_groups    text[] NOT NULL,         -- e.g. ['chest', 'triceps']
  secondary_muscles text[],
  equipment        text[] NOT NULL,         -- e.g. ['barbell', 'bench']
  difficulty       text CHECK (difficulty IN ('beginner', 'intermediate', 'advanced')),
  exercise_type    text CHECK (exercise_type IN ('strength', 'cardio', 'flexibility', 'balance')),
  location         text CHECK (location IN ('gym', 'home', 'both')),
  gender_target    text DEFAULT 'both' CHECK (gender_target IN ('male', 'female', 'both')),
  is_custom        boolean NOT NULL DEFAULT false,
  created_by       uuid REFERENCES auth.users(id) ON DELETE SET NULL,
  created_at       timestamptz NOT NULL DEFAULT now(),
  updated_at       timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX idx_exercises_muscles ON exercises USING gin(muscle_groups);
CREATE INDEX idx_exercises_equipment ON exercises USING gin(equipment);
```

**RLS:**
```sql
ALTER TABLE exercises ENABLE ROW LEVEL SECURITY;
-- Public exercises readable by all authenticated users
CREATE POLICY "exercises_read_public" ON exercises
  FOR SELECT USING (is_custom = false OR created_by = auth.uid());
-- Users can only insert/update/delete their own custom exercises
CREATE POLICY "exercises_write_own" ON exercises
  FOR ALL USING (is_custom = true AND created_by = auth.uid())
  WITH CHECK (is_custom = true AND created_by = auth.uid());
```

## Auto-Update Trigger

Applied to all tables with an `updated_at` column:

```sql
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- Applied per table, e.g.:
CREATE TRIGGER set_updated_at
  BEFORE UPDATE ON profiles
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

---

## Profile Auto-Creation Trigger

Creates a `profiles` row whenever a new user signs up:

```sql
CREATE OR REPLACE FUNCTION handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO profiles (user_id, locale)
  VALUES (NEW.id, 'ro');
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION handle_new_user();
```
